# Contributing to GSF

First off — thanks for taking the time to contribute to GSF.
We’re building a platform for founders who execute, not just talk.

## 🚀 How to Contribute

### 1. Fork the Repository

Click the "Fork" button on GitHub.

### 2. Clone Your Fork

git clone https://github.com/YOUR_USERNAME/GSF.git
cd GSF/GSF

### 3. Create a Feature Branch

git checkout -b feat/your-feature-name

### 4. Make Your Changes

* Follow the existing design system (colors, spacing, typography)
* Keep components modular and reusable
* Maintain TypeScript type safety
* Use Tailwind utility classes consistently

### 5. Commit Changes

Use clear commit messages:

* feat: add expert filtering
* fix: resolve booking bug
* ui: improve hero section layout

### 6. Push to GitHub

git push origin feat/your-feature-name

### 7. Open a Pull Request

* Describe what you built
* Add screenshots if UI-related
* Link related issues if applicable

---

## 🧠 Development Guidelines

### Code Style

* Use TypeScript strictly
* Prefer functional components
* Keep files small and focused
* Avoid unnecessary complexity

### UI/UX Principles

* Clean, premium, minimal
* Founder-first experience
* Fast interactions (no laggy flows)

### Folder Structure

Follow the existing structure:

* app/ → routes
* components/ → reusable UI
* landing/ → homepage sections
* ui/ → atomic components

## 🐛 Reporting Bugs

Open an issue with:

* Clear description
* Steps to reproduce
* Expected vs actual behavior
* Screenshots (if applicable)

## 💡 Suggesting Features

We welcome ideas that:
